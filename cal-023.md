# CAL-023 — Automatic Holiday and Birthday Event Generation

## Summary

Introduce system-generated calendar events for holidays and staff birthdays.

These events should be generated at runtime and integrated into the existing event pipeline without modifying layout or rendering systems.

---

## Goals

- Support automatic holiday events
- Support automatic birthday events based on staff data
- Keep system-generated events clearly distinguishable from manual events
- Preserve existing calendar architecture and rendering behavior

---

## Non-Goals

- No persistence of generated events
- No editing of generated events
- No UI toggles (initial implementation)
- No changes to calendar layout or lane logic
- No new modal patterns

---

## Architecture Plan

### Event Flow

manual events

- generated holiday events
- generated birthday events  
  → combined event list  
  → assignEvents.ts  
  → render

### Key Principle

Event generation must NOT happen in components.

---

## Data Modeling

### Event Source

Add a field to distinguish event origin:

```ts
type EventSource = 'manual' | 'holiday' | 'birthday';
```

All generated events must include:

```ts
source: 'holiday' | 'birthday';
```

---

## Holiday Events

### Requirements

- Support fixed-date holidays (e.g., July 4)
- Support variable-date holidays (e.g., Thanksgiving)
- Define a minimal initial set of holidays

### Implementation Options

- Static config file (recommended)
- Utility functions for calculating variable dates

---

## Birthday Events

### Requirements

- Add optional `birthdate` field to Staff type
- Support yearly recurrence
- Generate events from staff data

### Dependencies

- Staff type update
- Staff create/edit UI updates

---

## Generation Strategy

### Recommended Approach

- Create a dedicated utility (e.g., `autoEvents.ts`)
- Generate events based on:
  - visible date range OR
  - current calendar view window

### Output

Generated events should match the shape of existing events and be compatible with:

- assignEvents.ts
- CalendarDay.vue
- existing modal system

---

## Integration Point

Generated events should be merged with manual events at the point where the full event list is assembled before assignment.

Do NOT modify `assignEvents.ts` to generate events.

---

## Constraints

- No layout or lane changes
- No new modal patterns
- Maintain existing event rendering behavior
- Maintain avatar-first identity presentation where applicable
- Follow existing token styling and UI conventions

---

## Suggested Implementation Order

1. Add `birthdate` to Staff type
2. Update staff create/edit flows to support birthdays
3. Implement holiday generation utility
4. Implement birthday generation utility
5. Merge generated events into event pipeline
6. Validate rendering and assignment behavior

---

## Risks

- Scope creep due to UI changes (staff editing)
- Event type incompatibility if not aligned with existing models
- Incorrect placement of generation logic (must stay out of components)

---

## Future Enhancements

- Toggle visibility of generated events
- Allow editing or overriding generated events
- Configurable holiday sets
- Localization support
- Shared recurrence utilities

---

## Notes

- Keep implementation incremental and PR-driven
- Avoid introducing parallel systems
- Prefer extending existing utilities and patterns
